# TradingBrain 交易策略記憶（Claude 交接版）

## 文件目的

這份文件不是程式碼說明，也不是功能清單。

它的用途是把目前對 `TradingBrain` 的正確交易理解、策略哲學、風控邏輯與後續改造方向整理成一份可交接記憶，讓後續協作模型或工程師可以直接理解：

- 這套系統應該怎麼交易
- 哪些舊思路是錯的
- 目前真正應該堅持的核心原則是什麼
- 接下來怎麼改才會更像真正的加密貨幣程式交易系統

---

## 一、核心交易哲學

這套系統不應該再以「指標驅動 + 模板化 ATR 出場」作為主體。

正確方向應該是：

1. 先看市場結構
2. 再看 K 線確認
3. 指標只拿來輔助，不拿來主導
4. 止損不是固定比例
5. 止盈不是固定 ATR
6. 槓桿不是固定值，而是結果

這裡最重要的一句話是：

**結構是主體，指標是工具，槓桿是結果。**

指標可以告訴系統：

- 市場是否有趨勢
- 動能是否偏強或偏弱
- 是否接近過熱或過冷

但指標本身不是策略。

真正的策略必須回答：

- 什麼情境下可以進場
- 這筆交易的邏輯在哪裡失效
- 該在哪裡先收
- 剩下的部位如何讓它跑
- 這筆單最多願意冒多少風險

---

## 二、對「止損」的正確理解

### 1. 不是沒有止損，而是不能用太低級、太機械的止損

使用者明確表達過：

- 正常影線掃一下，不應該立刻出場
- 只要交易假設沒有真正失效，就不該太早被洗掉
- 這種「撐一下」不是凹單，而是要有規則地給市場呼吸空間

因此，止損應該分成兩層：

### 2. Soft Stop（軟失效）

這層是交易邏輯失效的判定，不是單純價格碰到某點就砍。

原則：

- wick 掃過不一定走
- 要看收盤有沒有真正失守
- 可以要求 `1m` 連續兩根收破，或 `5m` 一根完整收破

例子：

- 支撐區在 `70800 ~ 70860`
- 如果 `1m` 只是插到 `70770` 後收回來，不算失效
- 如果 `1m` 連續兩根收在 `70800` 下方，或 `5m` 一根收在 `70800` 下方，才算失效

### 3. Hard Stop（硬失效）

這層是災難保命線，主要用來防：

- 程式中斷
- API 異常
- 網路斷線
- 急跌急漲穿透
- 極端行情

Hard Stop 一定要存在，而且位置要比 Soft Stop 更遠。

Hard Stop 不是正常出場手段，而是防災機制。

### 4. 正確的止損邏輯

止損不應該先由 ATR 決定，再硬套到市場。

正確順序應該是：

1. 先找結構失效區
2. 再定 Soft Stop
3. 再定 Hard Stop
4. 最後才根據距離反推倉位

也就是說：

- 先決定哪裡錯
- 再決定可以開多大

不是反過來。

## 三、結構不是單點，而是區域

系統不應該把支撐、壓力、止損、止盈都理解成單一價位。

真正市場裡比較合理的做法是：

- 進場區
- 失效區
- 目標區

都應該是區域，不是一條線。

### BTC 特別需要更大的結構區

使用者明確指出：

- `BTC` 只放幾百 `U` 的結構區，很容易被正常波動掃掉

這個判斷是對的。

因此，對 `BTC 15m` 來說，不應該使用過窄的區域感。

合理理解應該是：

- `BTC 15m` 的結構區通常至少應該是幾百 `U` 到一千多 `U` 的寬度概念
- 不能把正常波動誤判成結構失效

重點不是固定多少 `U`，而是：

**結構區寬度必須符合該幣種、該時間框架、該波動環境。**

## 四、市場應分成三種，而不是只分趨勢 / 盤整

這套系統最重要的修正之一，是要接受「過渡市場」的存在。

### A. 趨勢市場

特徵：

- 高低點明顯朝同方向推進
- `4h / 1h` 方向一致
- 回踩會被接住或壓住

適合：

- `trend following`
- `breakout`

### B. 區間市場

特徵：

- 上下邊界清楚
- 反覆來回
- 沒有明確延續結構

適合：

- `mean reversion`

### C. 過渡市場

這是目前系統曾經最缺的類型。

特徵：

- 有方向傾向，但不夠乾淨
- 不是純趨勢，也不是標準區間
- 很多幣長時間卡在這種狀態

處理方式：

- 不是全部封死
- 可以做
- 但只能做高品質單
- 而且風險要下降

如果沒有這個分類，系統就很容易出現：

- 所有市場都不夠漂亮
- 所有策略都被 gate 擋掉
- 最後連續兩天沒單

## 五、進場不該是「指標成立就進」，而該是「結構 + K 線確認」

這是整套交易邏輯最核心的改造方向。

### 1. Trend Following

不該看到 EMA 交叉就追。

正確流程應該是：

1. `4h / 1h` 先提供方向背景
2. `15m` 出現合理 pullback 到結構區
3. `1m / 5m` 出現 reclaim、engulfing、higher low / lower high 之類的 K 線確認
4. 才進場

也就是：

- 趨勢是背景
- 拉回是機會
- K 線是扳機

例子：

- `BTC 4h/1h` 偏多
- `15m` 回踩前高轉支撐區 `70800 ~ 70860`
- 沒有直接跌破
- `1m` 出現 bullish engulfing，重新站回 `70880`
- 這時候做多

這種單比追在 `71100` 的 EMA 黃金交叉合理很多。

### 2. Breakout Retest

突破單最常犯的錯，就是把「碰一下」當成突破。

正確流程應該是：

1. 定義明確箱體、前高、前低、swing high / low
2. 真正突破結構，而不是只突破 band
3. 等 retest
4. retest 撐住，再由 `1m / 5m` 做確認

例子：

- `ETH` 箱體高點在 `2200`
- `15m` 收盤站上 `2200`
- 不馬上追
- 等 `1m / 5m` 回踩 `2196 ~ 2202`
- 回踩沒破，又重新站上 `2205`
- 再進多

這才是真正的 breakout retest。

### 3. Mean Reversion

均值回歸不是 RSI 低就買。

正確流程應該是：

1. 價格來到區間邊界
2. 出現反轉 K 線
3. 小級別重新收回關鍵價位
4. 才進場

例子：

- `XRP` 區間下緣在 `1.35`
- 價格跌到 `1.352`
- 出現長下影 + 吞噬
- `1m` 重新收回 `1.356`
- 才進多

這樣才叫均值回歸，不是看到超賣就賭。

## 六、止盈應該看結構區，不該只靠 ATR

使用者已明確指出，單純用固定 `%` 或固定 ATR 模板做 TP，不符合真實交易邏輯。

這個判斷是正確的。

### 1. Trend Following 止盈

- `TP1`：最近一個明顯阻力 / 支撐區
- `TP2`：下一個 `1h` 結構位
- 剩餘部位：讓它沿著更高低點或更低高點去跑

### 2. Breakout 止盈

- `TP1`：箱體高度投射的第一目標，或最近供需區
- `TP2`：下一個更高級別結構區
- 不要太早保本，不然很容易被正常回踩洗掉

### 3. Mean Reversion 止盈

- `TP1`：回到中線 / 均值
- `TP2`：回到區間另一半或另一側之前先收大部分
- 這種單不應該抱成趨勢單

## 七、目前最常見的錯誤之一：太快保本，太快 trailing

這不是單純「止損太近」而已。

系統另一個經典錯誤是：

- 方向其實沒錯
- 只是正常回踩一下
- 但因為 TP1 太近
- 很快保本
- 很快 trailing
- 最後被自己洗掉

因此，應該這樣調整：

### 趨勢 / 突破單

- TP1 先減少風險，但不要立刻把止損移回 entry
- 更合理做法是移到新的結構低點 / 高點下方
- trailing 應該看 swing low / swing high
- 不應該只用 ATR 緊貼

### 均值回歸單

- 可以比較快保本
- 因為本來就不是要抱長波段

也就是：

- 趨勢單、突破單要給市場空間
- 均值回歸要快

## 八、槓桿不是主變數，風險才是主變數

使用者明確要求：

- 帳戶大約 `5000U`
- 目標是每天盈利 `50U ~ 200U`

這代表系統不能再用固定槓桿思維。

正確順序應該是：

1. 先決定這筆單最多願意虧多少 `U`
2. 再根據止損距離、結構品質、市場型態、當日狀態去反推倉位
3. 槓桿是算出來的結果，不是預設值

### 5000U 帳戶的風險分級

#### A級單

條件：

- 結構清楚
- 多時間框架支持
- K 線確認乾淨
- RR 漂亮

風險：

- `1.5% ~ 2.0%`
- 約 `75U ~ 100U`

#### B級單

條件：

- 條件不錯，但不是最漂亮

風險：

- `0.8% ~ 1.2%`
- 約 `40U ~ 60U`

#### C級單

條件：

- 邊緣單
- 過渡市場
- 確認不夠漂亮

風險：

- `0.3% ~ 0.6%`
- 約 `15U ~ 30U`

這樣做的意義是：

- 市場差時仍能有小單
- 市場漂亮時才放大火力

## 九、日內節奏管理

既然目標是 `5000U -> 每天 50~200U`，就不能只看單筆邏輯，還要看當日節奏。

### 盈利日

- `+80U`：只做 A 級單
- `+150U`：接近收工
- `+200U`：直接停手

### 虧損日

- `-60U`：降一級風險
- `-100U`：只做最漂亮的單
- `-150U`：停手

這樣的目的是：

- 獲利日保住成果
- 虧損日不要失控

## 十、具體交易案例

### 案例 1：BTC Breakout Retest Long

假設：

- `4h/1h` 偏多
- `15m` 箱體上沿在 `70800`
- retest 成功後想做多

做法：

- 進場：`70860`

Soft Stop：

- 連續兩根 `1m` 收在 `70790` 下方
- 或一根 `5m` 收在 `70780` 下方

Hard Stop：

- `70690`

TP：

- `TP1 = 71120 ~ 71180`，先出 `25%`
- `TP2 = 71450`，再出 `35%`
- 剩餘部位沿 `5m higher low` trailing

這代表：

- 影線掃一下不理
- 真正收破才承認交易失敗
- 極端行情還是有保命線

### 案例 2：NEAR Mean Reversion Long

假設：

- `NEAR` 區間下沿在 `1.350`
- 價格打到 `1.352`
- `15m` 出現下影線
- `1m` reclaim `1.356`

做法：

- 進場：`1.358`

Soft Stop：

- `1m` 連續兩根收在 `1.349` 下方

Hard Stop：

- `1.344`

TP：

- `TP1 = 1.378`
- `TP2 = 1.398 ~ 1.405`

補充：

- 如果這種均值回歸單在 `6~8` 根 `1m` 內都彈不起來，應該考慮提早走
- 因為均值回歸不該拖成趨勢單

## 十一、目前最合理的改造順序

### 第一優先

重做止損 / 出場哲學：

- `soft stop + hard stop`
- 不再只用單一價位
- TP1 後不要太快保本
- trailing 改成結構追蹤

### 第二優先

重做 breakout 與 trend 的進場方式：

- 趨勢單改成 pullback continuation
- 突破單改成真正結構 breakout + retest
- 均值回歸只在 range edge 做

### 第三優先

槓桿與風險分級：

- A / B / C setup grading
- 動態風險預算
- 日內節奏控制

## 十二、對整套系統最終的理解

如果這套系統真的要像專業交易系統，而不是像模板化指標機器，那它不應該被設計成：

- RSI 到幾就買
- ATR 幾倍就砍
- TP1 一到就保本
- 全部策略共用一種止盈止損語言

它應該被設計成：

- 先看結構
- 再看 K 線
- 指標只輔助
- 止損分軟硬
- 止盈看區域
- 倉位由結構止損距離反推
- 槓桿只是結果，不是目標

這份記憶是後續所有優化的核心基礎。

如果後續程式設計與這份文件衝突，應優先重新審視程式，而不是先懷疑這份交易哲學。
