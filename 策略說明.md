# 策略說明

## 1. 現在這套系統在做什麼

TradingBrain V9 是一套以 Binance Futures 為核心的自動交易系統。  
它不是單純靠幾個技術指標數值亮燈就下單，而是用：

- 市場結構
- 多時間框架方向
- K 線確認
- 分策略風控
- 雙層止損

去完成一筆交易。

## 2. 核心交易哲學

這套系統的正確理解不是：

- RSI 到多少就買
- ATR 幾倍就砍
- TP1 一到就直接保本

而是：

- 先看結構
- 再看 K 線
- 指標只輔助
- 槓桿是結果，不是起點

## 3. 市場分類

### 3.1 趨勢盤

特徵：

- 高低點持續往同一方向推進
- `4h / 1h` 方向較一致

適合策略：

- `trend_following`
- `breakout_retest`

### 3.2 區間盤

特徵：

- 上下邊界清楚
- 缺乏乾淨延續

適合策略：

- `mean_reversion`

### 3.3 過渡盤

特徵：

- 有方向傾向，但不夠乾淨
- 不是純趨勢，也不是標準區間

處理方式：

- 可以做，但只做高品質單
- 風險降檔

## 4. 多時間框架分工

- `4h`
  - 大方向背景
- `1h`
  - 方向過濾與補充信心
- `15m`
  - setup 產生
- `1m`
  - 最終進場扳機

## 5. 三個策略

### 5.1 Trend Following

定位：

- 順勢回踩續行

做法：

- 不是追剛噴出去的交叉
- 要等 `15m` 回踩到合理結構區
- 再等 `1m / 5m` 重新站回或跌回

### 5.2 Breakout Retest

定位：

- 真正結構突破後，等回踩確認再進

做法：

- 不追突破當下
- 先建立 `pending`
- 等 `retest`
- 成功後才 `confirmed`

### 5.3 Mean Reversion

定位：

- 區間邊界回歸

做法：

- 不在強趨勢裡亂抄底摸頭
- 只在 range edge 附近做
- 一定要有反轉 K 線或 reclaim 確認

## 6. 止損與止盈

### 6.1 Soft Stop

- 代表交易邏輯失效
- 需要收盤確認
- 不是 wick 掃一下就出場

### 6.2 Hard Stop

- 代表災難保護
- 用來防斷線、暴跌暴漲、極端行情
- 會比 soft stop 更遠

### 6.3 結構區域

系統會記錄：

- `soft_stop_loss`
- `hard_stop_loss`
- `stop_zone_low / stop_zone_high`
- `tp1_price / tp2_price / tp3_price`
- `tp1_zone_* / tp2_zone_* / tp3_zone_*`

### 6.4 三策略出場風格

- `trend_following`
  - 保留較大的趨勢呼吸空間
- `breakout_retest`
  - 以 retest 失敗與結構失效為主
- `mean_reversion`
  - 快進快出，不拖成趨勢單

## 7. 風險與槓桿

核心觀念：

- 槓桿不是固定值
- 先決定願意虧多少，再反推倉位

以 `5000U` 帳戶為例，系統更偏向：

- `A級單`
  - 高品質結構 + 乾淨 K 線 + MTF 支持
- `B級單`
  - 條件不錯，但不是最好
- `C級單`
  - 邊緣單、過渡市場、縮小火力

## 8. Telegram 通知規則

目前系統會在以下情況送 Telegram：

- 系統啟動
- 每小時監控快報
- 開倉
- TP / SL / trailing 出場
- 每日報告
- 風控攔截

如果是測試訊息，會明確標示為：

- `🧪 TradingBrain V9 交易測試`

## 9. Daily Report 與 Log

系統會持續記錄：

- analysis logs
- trades logs
- error logs
- daily reports
- 自動分析報告

每日報告與自動分析目前都以 `Asia/Bangkok` 為主時區。

## 10. 現在這版的重點

這版不再是早期那種：

- 單一 ATR 模板止損
- 單線 stop / take profit
- 舊版通知管道

而是：

- 結構優先的交易管理
- 雙層止損
- 三策略拆分
- Telegram 通知
- 可研究的 logging

一句話總結：

這套系統現在的核心不是「預測市場」，而是：

- 在合理市場背景下
- 找到合理結構位置
- 等 K 線確認
- 用對應策略的方式進出場
- 嚴格控制風險
