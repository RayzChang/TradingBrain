# 給出資人 — 你只要做這兩件事

> 你是出資人，Agent（我）是實際作者。目標：讓這個案子賺到錢。  
> 下面是你**具體要做的節奏**，其餘我來。

---

## 一、執行程式

在專案目錄執行：

```bash
venv\Scripts\python main.py
```

讓它常駐跑（可縮小視窗或背景跑）。程式跑著就會**每 60 分鐘**自動發一則 LINE「監控快報」給你。

---

## 二、每幾則回報來讓 Agent 改一次？

**建議：每收到 3 則監控快報（約 3 小時），就來 Cursor 一次。**

要做的事：

1. 打開 Cursor，開這個專案，跟 Agent 對話。
2. 把**最近 3 則 LINE 監控快報的內容**貼到對話裡（複製貼上即可）。
3. 說一句話，例如：
   - 「根據這些回報改策略」
   - 「這 3 則是過去約 3 小時的，請根據這個調整」

我就會根據你貼的數據（今日損益、今日筆數、未平倉）**只改專案裡的「大腦」**（`data/brain_state.json`），不再改寫死的程式碼：

- **若連續都是「今日筆數 0、未平倉 0」** → 我放寬大腦參數（例如 ADX 門檻更低、RSI 區間更寬、否決放寬），讓程式更容易出單。**程式每 15 分鐘會自動讀取大腦，你不用重啟 main。**
- **若有單但虧** → 我收緊大腦（提高門檻、或關閉否決放寬）。
- **若有單且賺但未達 50~100 U/日** → 我微調大腦參數往提高期望值方向走。
- **有賺要收斂**：系統已支援「每日獲利達標就暫停開新倉」鎖利；若你曾一度大賺卻又吐回去，會依風控的 `daily_profit_target` 在達標後不再開新單，只讓既有倉位跑完。

改完我會告訴你「大腦已更新」，**通常不需要重啟**；你只要繼續跑、等下一輪 3 則回報再貼給我即可。

---

## 為什麼是「每 3 則」？

- 1 則 = 約 1 小時，樣本太少。
- 3 則 = 約 3 小時，足以看出「是持續 0 筆還是有在動」。
- 你若很忙，也可以改成**每 6 則（約 6 小時）**或**每天一次**把當天收到的快報貼給我也行；節奏你定，但要有「固定把回報貼給 Agent」的習慣，我才會有數據可以改。

---

## 若昨天跑一整夜、什麼都沒有

那就**從現在開始**：main 跑起來 → 等 3 則快報（約 3 小時）→ 把那 3 則貼到 Cursor 說「根據這些改策略」。  
即使 3 則都是「今日筆數 0」，那也是數據：我會依「長期 0 單」去改（放寬出單條件、檢查執行層），不會沒有 log 就不改。
