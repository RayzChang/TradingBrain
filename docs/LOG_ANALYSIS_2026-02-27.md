## 2. 做得不錯的地方

1. **風控有在擋**  
   Log 大量出現「風控攔截: XXX - 已達最大持倉數 3 >= 3」，代表最大持倉數有生效，沒有爆倉位。

2. **否決有在擋**  
   「Signal VETOED: … 絞肉機行情偵測中，暫停開單」有出現，代表震盪市有被擋掉一部分，不是全部放行。

3. **Paper 流程正確**  
   - 開倉：`paper 開倉完成: trade_id=…` 寫入 DB、發 LINE。  
   - 平倉：雖有「平倉下單失敗」（因 paper 不真的打交易所），但隨後「position_check 已平倉: … TAKE_PROFIT/STOP_LOSS pnl=…」有執行，DB 正確結算並發 LINE。

4. **停損有執行**  
   該觸發的 STOP_LOSS 都有在 log 裡出現並寫入 DB，沒有漏平。

5. **有賺錢的單**  
   6 筆 TP 顯示策略在對的時機有賺到，不是全盤皆輸。

## 3. 做得很糟／要改的地方

1. **勝率偏低、總損益為負**  
   26% 勝率、淨 -1644 U，代表止損筆數與單筆虧損偏高，或進場時機常落在震盪/不利結構。

2. **同標的反覆同向開倉**  
   - AVAX LONG 多次（有 TP 也有多次 SL）。  
   - DOT 連續 LONG 後連續 SL。  
   - ETH 多筆 LONG/SHORT 交錯，部分很快 SL。  
   → 建議後續：同一 symbol 短時間內已有同向倉時，可考慮「不再加碼同向」或提高門檻（需策略或風控層支援，大腦可先註記）。

3. **趨勢策略在震盪時開單**  
   多筆 `trend_following` 開 LONG 後很快觸發 STOP_LOSS（如 BTC、BNB、DOT、SOL），顯示在非明確趨勢時仍出信號。  
   → 可藉大腦 **提高 adx_min** 或 **skip_on_chop: true** 減少這類進場。

4. **Log 噪音**  
   Paper 模式下「平倉下單失敗」是預期（未打交易所），但每筆都打 WARNING 會干擾排查；可考慮 paper 時不呼叫交易所 close，只做 DB+LINE（程式改動，非本輪大腦範圍）。

## 5. 收斂與鎖利（交易大腦該做的事）

- **問題**：曾一度 +5000U（約翻倍）卻未收斂，繼續開單導致回吐變虧。  
- **已實作**：風控層 `daily_limits` 現在會讀取 `daily_profit_target`（config/risk_defaults 或 DB）：當「今日已實現獲利 ≥ 當日起始權益 × daily_profit_target」時，**禁止再開新倉**，只讓既有倉位跑完，鎖利。  
- **設定建議**：  
  - 想早收：`daily_profit_target: 0.03`～`0.06`（3%～6% 就停）。  
  - 想翻倍才收：`daily_profit_target: 1.0`（100%，今日獲利 ≥ 當日起始權益才停）。可透過 API 或 DB `risk_params` 寫入。

## 6. 提高勝率的方向（26% → 值得留用）

| 方向 | 做法 |
|------|------|
| 少在震盪開單 | 大腦 `skip_on_chop: true`（已開）、必要時提高 `adx_min`（如 16～18） |
| 趨勢單更謹慎 | 僅在明確趨勢出信號，避免 trend_following 在 chop 開 LONG 被洗掉 |
| 同標的少重複 | 後續可加「同 symbol 同向冷卻」或最大持倉 per-symbol |
| 鎖利不貪多 | 使用 `daily_profit_target`，達標就停開新倉，避免賺了又吐回去 |

勝率若長期 < 35% 且期望值為負，大腦應優先「收緊進場」與「獲利鎖利」，再考慮加新指標（如斐波那契、頭尖頂）。
