## 系統總覽

TradingBrain 是一套加密貨幣 USDT 合約自動交易系統，運作邏輯如下：

```
WebSocket 即時 K 線 → 技術分析 → 策略信號 → 否決引擎 → 風控 → 下單 → 通知
```

**最簡單的啟動方式（推薦）：**

```bash
python launcher.py
```

瀏覽器會自動開啟控制台，你可以在介面上完成所有設定和操作。

**或者手動操作：**
1. 設定 `.env`（API Key、LINE 通知等）
2. 執行 `python setup_testnet.py`（設定槓桿和保證金模式）
3. 執行 `python main.py`（啟動交易引擎）

| 區塊 | 功能 |

| 🧠 交易大腦 | 啟動/停止交易引擎。按「▶ 啟動」開始交易，按「⏹ 停止」安全關閉 |

### ⚙️ 設定（環境變數）

這個分頁讓你直接在介面上修改 `.env` 檔案中的所有設定，不用手動開文字編輯器。

#### 🔑 幣安 API

| 設定 | 說明 | 安全提醒 |
|------|------|----------|
| **API Key** | 你在幣安交易所生成的 API Key | ⚠️ 只需開啟「合約交易」權限，**不要開啟提領權限** |
| **API Secret** | 與 API Key 配對的密鑰 | ⚠️ 絕對不要分享給任何人 |
| **Testnet 模式** | 選擇連線的交易所環境 | 上線前務必在 Testnet 充分測試 |
| **放寬否決 (Testnet)** | 是否在 Testnet 時放寬否決引擎 | 見下方詳細說明 |

**Testnet 模式 詳細說明：**

| 選項 | 連線到 | 用途 |
|------|--------|------|
| ✅ 是 (Demo) | `testnet.binancefuture.com` | **推薦新手使用。** 使用虛擬資金交易，不會有任何真實損失。幣安提供免費的 Demo 帳戶 |
| ❌ 否 (實盤) | `fapi.binance.com` | **正式交易！** 使用你帳戶裡的真實 USDT。一旦選擇此模式，所有交易都是真金白銀 |

**放寬否決 (RELAX_VETO_ON_TESTNET) 詳細說明：**

否決引擎會根據市場情緒（恐懼貪婪指數、資金費率）阻擋交易信號。但在 Testnet 環境，這些數據來自真實市場，可能會導致 Demo 帳戶長時間不開單。

| 選項 | 效果 |
|------|------|
| ✅ 是 | 恐懼貪婪指數和資金費率的否決被禁用，只保留「爆倉異常」和「絞肉機行情」的否決。**讓 Demo 更容易開單驗證流程** |
| ❌ 否 | 所有否決條件都啟用，與實盤完全一致。**更真實但可能很久不開單** |

> 💡 **建議：** Demo 測試時選「是」，正式上線前切回「否」測試一天，確認否決邏輯是你想要的。

---

#### 💰 交易設定

| 設定 | 說明 | 建議 |
|------|------|------|
| **交易模式** | 見下方詳細說明 | 新手從 Paper 開始 |
| **初始本金 (USDT)** | 你投入交易的本金數額 | 風控按這個金額計算倉位大小，與交易所實際餘額無關 |
| **保證金模式** | 全倉 (CROSSED) 或 逐倉 (ISOLATED) | 見下方詳細說明 |
| **槓桿倍率** | 1x ~ 10x | 見下方詳細說明 |

**交易模式 (TRADING_MODE) 詳細說明 — ⚠️ 最重要的設定：**

| 模式 | 效果 | 風險 |
|------|------|------|
| 📝 **Paper (僅模擬)** | 策略正常運行、信號正常產生，但**不會向交易所發送任何訂單**。所有交易都只是寫入本地資料庫的模擬紀錄。你可以在儀表板上看到模擬損益 | **零風險** — 適合剛開始使用、驗證策略邏輯 |
| 🔴 **Live (實際下單)** | 策略信號通過風控後，會**真的向交易所發送市價單**。訂單會出現在你的幣安帳戶裡 | **有真實資金風險** — 確保你理解所有設定後再啟用 |

> ⚠️ **重要：** 即使設定為 Live 模式，也需要 Testnet=否 才會打到真正的交易所。如果 Testnet=是 + Live，訂單會發到 Demo 交易所（安全）。

**組合對照表：**

| Testnet | 交易模式 | 實際效果 |
|---------|----------|----------|
| ✅ 是 | Paper | 完全模擬，不發訂單 |
| ✅ 是 | Live | **在 Demo 交易所下單**（推薦的測試方式） |
| ❌ 否 | Paper | 連實盤資料但不下單 |
| ❌ 否 | Live | **⚠️ 真實交易！使用真金白銀** |

**保證金模式 (MARGIN_TYPE) 詳細說明：**

| 模式 | 說明 | 適用場景 |
|------|------|----------|
| **全倉 (CROSSED)** | 所有持倉共享帳戶的全部餘額作為保證金。某個倉位虧損時，其他倉位的利潤可以幫忙補充保證金，降低被強制平倉的機率 | 適合想要**多持倉互補**的玩家 |
| **逐倉 (ISOLATED)** | 每個倉位有獨立的保證金。虧損最多只會虧到該倉位的保證金，不會波及其他倉位 | 適合想要**嚴格控制單倉風險**的玩家 |

> 💡 **建議：** 新手用全倉 (CROSSED)，搭配系統的風控上限，風險已經被控制住。

**槓桿倍率 (DEFAULT_LEVERAGE) 詳細說明：**

| 倍率 | 說明 | 風險 |
|------|------|------|
| **1x** | 無槓桿，等於現貨交易 | 最低風險 |
| **2x** | 10% 漲幅 = 20% 利潤 | 低風險 |
| **3x** | 系統預設推薦 | 中等風險 |
| **5x** | 已經算高，新手不建議 | 較高風險 |
| **10x** | 極高，10% 下跌就虧完保證金 | ⚠️ 極高風險 |

> ⚠️ **注意：** 系統的風控是基於本金的 3% 計算倉位大小。槓桿越高，實際開倉的合約面值越大，但投入的保證金不會超過風控規定。槓桿主要影響的是「多少漲跌幅會觸發爆倉」。

### 使用流程建議

```
1. python launcher.py          ← 開啟控制台
2. 設定分頁 → 填入 API Key     ← 第一次使用才需要
3. 設定分頁 → 確認模式          ← Testnet=是, 交易模式=Live (推薦)
4. 設定分頁 → 按「儲存設定」
5. 控制台 → 按「執行設定」       ← 設定交易所的槓桿和保證金

7. 等綠燈亮起後 → 按「開啟儀表板」 ← 查看交易狀態
```

## 根目錄檔案

### `main.py` — 系統入口（最重要的檔案）

**作用：** 整個交易機器人的啟動點。打開它等於啟動整台機器。

**做了什麼：**
1. 初始化日誌、資料庫、風控參數
2. 啟動 WebSocket 連線，接收 10 個幣種的即時 K 線數據
3. 用 REST API 預載歷史 K 線（確保 MTF 分析有足夠數據）
4. 註冊所有定時任務（資金費率、爆倉監控、心跳、每日報告等）
5. 每當一根 K 線收盤，觸發技術分析 → 策略信號 → 否決 → 風控 → 下單
6. 在背景啟動 FastAPI（Port 8888），供儀表板使用
7. 發送 LINE 啟動通知

**核心類別：** `TradingBrain` — 交易系統主控制器，包含所有子系統的串接邏輯。

---

### `setup_testnet.py` — Testnet 初始設定工具

**作用：** 在正式開始交易之前，幫你做好交易所端的設定。

**做了什麼：**
1. 查詢你 Testnet 帳戶的 USDT 餘額和當前持倉
2. 依 `.env` 設定，對 10 個監控幣種設定保證金模式（全倉 CROSSED / 逐倉 ISOLATED）
3. 對 10 個監控幣種設定槓桿倍率

**何時用：** 第一次跑機器人之前、或想改槓桿/保證金模式時跑一次即可。

---

### `.env` / `.env.example` — 環境變數

**作用：** 存放所有敏感的設定（API Key、密碼等），不會上傳 Git。

> 💡 **推薦：** 使用控制台啟動器的「設定」分頁來修改這些設定，不需要手動編輯檔案。每個設定的詳細說明請參考上方 [控制台啟動器 → 設定](#⚙️-設定環境變數) 章節。

---

### `requirements.txt` — Python 套件清單

**作用：** 安裝所有 Python 依賴。用 `pip install -r requirements.txt` 一鍵安裝。

## config/ — 全局配置

### `settings.py` — 全局配置管理中心

- LINE 設定

## core/analysis/ — 技術分析引擎

這是系統的「眼睛」— 負責看懂市場在發生什麼。

### `engine.py` — 分析引擎入口

**作用：** 技術分析的統一入口，串接所有分析子模組。

**做了什麼：**
- `analyze_single()` — 單一時間框架的完整分析（指標 + 背離 + K 線型態 + 斐波那契 + 絞肉機偵測）
- `analyze_full()` — 多時間框架完整分析（15m + 1h + 4h 聯合分析）
- `get_analysis_snapshot()` — 產生精簡的分析摘要（供 API 和策略使用）

---

### `indicators.py` — 技術指標計算

**作用：** 計算所有核心技術指標，一次算完。

**計算的指標：**

| 指標 | 用途 |
|------|------|
| RSI (14) | 超買（>70）/ 超賣（<30）判定 |
| MACD (12/26/9) | 趨勢動能、交叉信號、背離偵測 |
| 布林帶 (20, 2σ) | 波動率通道、均值回歸信號 |
| EMA (9/21/50/200) | 趨勢方向判定（多頭排列/空頭排列） |
| SMA (20/50/200) | 均線支撐阻力 |
| ATR (14) | 波動率度量，動態止損 |
| ADX (14) | 趨勢強度（≥20=有趨勢，<20=無趨勢） |
| OBV | 量能趨勢確認 |
| Stochastic RSI | 更靈敏的超買超賣 |
| VWAP | 機構級均價參考 |

---

### `candlestick.py` — K 線型態辨識

**作用：** 辨識 K 線圖上的經典型態，用純 Python 計算（不依賴外部庫）。

**支援的型態：**

| 型態 | 方向 | 說明 |
|------|------|------|
| 十字星 Doji | 中性 | 實體極小，市場猶豫不決 |
| 錘子線 Hammer | 看漲 | 長下影線，底部反轉信號 |
| 倒錘子 Inverted Hammer | 看跌 | 長上影線，頂部反轉信號 |
| 看漲吞噬 Bullish Engulfing | 看漲 | 陽線完全包裹前一根陰線 |
| 看跌吞噬 Bearish Engulfing | 看跌 | 陰線完全包裹前一根陽線 |
| 三白兵 Three White Soldiers | 看漲 | 連三陽線，強烈上升信號 |
| 三黑鴉 Three Black Crows | 看跌 | 連三陰線，強烈下降信號 |
| 晨星 Morning Star | 看漲 | 三K組合，下跌轉折 |
| 夜星 Evening Star | 看跌 | 三K組合，上漲轉折 |

---

### `divergence.py` — 背離偵測

**作用：** 偵測價格與指標之間的「不同步」，這通常是趨勢即將反轉的信號。

**偵測類型：**
- **看漲背離（Regular Bullish）：** 價格創新低，但 RSI/MACD 不創新低 → 可能反轉上漲
- **看跌背離（Regular Bearish）：** 價格創新高，但 RSI/MACD 不創新高 → 可能反轉下跌

**使用方法：** 用 swing high/low 偵測法，回溯 60 根 K 線，避免雜訊。

---

### `fibonacci.py` — 斐波那契回撤/擴展

**作用：** 計算關鍵的價格支撐/阻力位，協助判斷進出場位置。

**計算的位階：**
- 回撤位：0%, 23.6%, 38.2%, 50%, 61.8%, 78.6%, 100%
- 擴展位：100%, 127.2%, 161.8%, 200%, 261.8%

**輸出：** 最近的支撐位和阻力位距離當前價格多遠。

---

### `multi_timeframe.py` — 多時間框架分析 (MTF)

**作用：** 確保信號與大趨勢一致。核心原則：**高時間框架決定方向，低時間框架決定進場。**

**做了什麼：**
1. 分析 1d / 4h / 1h / 15m 各時間框架的趨勢方向
2. 判定是否「趨勢一致」（全看多/全看空/衝突）
3. 產出推薦方向（LONG / SHORT / None）和信心分數
4. HTF RSI 確認：做多時確認 4h/1h RSI 不超買，做空時確認不超賣

**重要性：** 信號如果與大時間框架方向衝突，會被直接過濾掉。

---

### `chop_detector.py` — 絞肉機偵測

**作用：** 偵測市場是否處於「絞肉機行情」— 價格劇烈震盪但沒有方向，頻繁觸發止損。

**偵測項目（各佔分數）：**

| 項目 | 權重 | 條件 |
|------|------|------|
| ADX 趨勢弱 | 30% | ADX < 20 |
| 影線佔比高 | 25% | 影線佔比 > 65% |
| 價格原地震盪 | 25% | 位移率 < 15% |
| 頻繁假突破 | 20% | 假突破 ≥ 4 次 |

**行為：** 分數 ≥ 0.5 判定為絞肉機，系統暫停開新倉 30~60 分鐘。

## core/strategy/ — 交易策略

這是系統的「大腦」— 決定何時買、何時賣。

### `base.py` — 策略基類

**作用：** 定義所有策略共用的介面和流程。

**核心機制（v3 改版）：**
1. **市場狀態偵測** — 用 ADX 判定當前是「趨勢」還是「震盪」
   - ADX ≥ 20 → 趨勢狀態 → 只用趨勢追蹤 + 突破策略
   - ADX < 20 → 震盪狀態 → 只用均值回歸策略
2. **MTF 方向過濾** — 信號必須與多時間框架推薦方向一致
3. **HTF RSI 確認** — 不在極端位做逆勢交易

---

### `trend_following.py` — 趨勢追蹤策略

**適用市場：** ADX ≥ 20（趨勢狀態）

**進場邏輯：** EMA 交叉 + ADX 趨勢強度 + K 線型態確認 + 斐波那契位確認 + 背離偵測

**30天回測：** 124 筆，50.8% 勝率，+3,452U

---

### `breakout.py` — 突破策略

**適用市場：** ADX ≥ 20（趨勢狀態）

**進場邏輯：** 布林帶突破 + 成交量增加 1.5 倍 + ADX 連續上升 + MACD 動能擴張

**30天回測：** 126 筆，55.6% 勝率，+4,601U

---

### `mean_reversion.py` — 均值回歸策略

**適用市場：** ADX < 20（震盪狀態）

**進場邏輯：** 價格觸及布林帶上下軌 + RSI 超買超賣 + K 線反轉型態確認

**狀態：** 在趨勢盤為主的行情中會被市場狀態偵測自動關閉。

---

### `signal_aggregator.py` — 信號聚合器

**作用：** 多策略的「交通指揮」，統一管理所有策略產生的信號。

**做了什麼：**
1. 執行所有策略，收集候選信號
2. **衝突解決** — 同一幣種同時有 LONG 和 SHORT 信號時，只保留最強的
3. **同標的冷卻** — 同一幣種同方向 2 小時內不重複開倉
4. 將信號送入否決引擎過濾
5. 記錄所有信號（含被否決的）到資料庫

---

### `coin_screener.py` — 幣種篩選

**作用：** 定期掃描市場，篩選出適合交易的幣種（用量能和波動率排名）。

---

## core/execution/ — 交易執行

這是系統的「手」— 負責實際下單。

### `binance_client.py` — 幣安 API 客戶端

**作用：** 與幣安交易所的 REST API 溝通，所有請求都經過 HMAC SHA256 簽名。

**功能：**
- 查詢 USDT 餘額
- 查詢持倉
- 市價開倉
- 掛止損單（STOP_MARKET）
- 掛止盈單（TAKE_PROFIT_MARKET）
- 市價平倉（reduceOnly）
- 設定槓桿、保證金模式
- 自動限流（遵守幣安 API 限制）

---

### `execution_engine.py` — 執行引擎

**作用：** 串接風控結果和幣安下單。

**三種模式：**

| 模式 | 條件 | 行為 |
|------|------|------|
| Paper | `TRADING_MODE=paper` | 僅模擬成交寫 DB，不打交易所 |
| Testnet | `BINANCE_TESTNET=true` + Key 已設 | 在 Demo 交易所下單 |
| 實盤 | `BINANCE_TESTNET=false` + `TRADING_MODE=live` | 在正式交易所下單 |

**下單流程：** 設定槓桿 → 市價開倉 → 掛止損 → 掛止盈 → 寫入 DB → 發 LINE 通知

---

### `position_manager.py` — 持倉管理

**作用：** 定期檢查現有持倉的止損/止盈狀態，觸及則平倉。也負責同步交易所持倉到本地 DB。

---

## core/risk/ — 風險管理

這是系統的「安全帶」— 防止你虧太多。

### `risk_manager.py` — 風控核心

**作用：** 統一風控入口，依序執行以下檢查：

| 順序 | 檢查 | 說明 |
|------|------|------|
| 1 | 每日熔斷 | 今天已虧超過 6% → 停止開單 |
| 2 | 連虧冷卻 | 連續虧 N 筆 → 暫停一段時間 |
| 3 | 持倉數上限 | 已持有 3 個倉位 → 不再開新倉 |
| 4 | 倉位計算 | 根據餘額和風險比例計算下單數量 |
| 5 | 止損止盈 | 用 ATR 計算動態止損（1.5 ATR）和止盈（4 ATR） |

全部通過才會下單。

---

### `position_sizer.py` — 倉位計算器

**作用：** 計算每筆交易應該投入多少錢。公式基於「每筆最多虧損本金的 3%」。

---

### `stop_loss.py` — 止損止盈計算

**作用：** 用 ATR（平均真實波動幅度）動態計算止損和止盈價位。

| 參數 | 預設 | 說明 |
|------|------|------|
| 止損 | 1.5 ATR | 給足空間不被洗出去 |
| 止盈 | 4.0 ATR | 讓利潤跑遠 |
| 部分止盈 | 2.0 ATR | 到了先走 50% |
| Trailing Stop | 1.5% | 剩餘 50% 跟蹤最高價 |

---

### `daily_limits.py` — 每日熔斷

**作用：** 監控每日損益，超過停損線就全天停止開單。預設 6%（5000U 本金 = 虧 300U 即停）。

---

### `cooldown.py` — 連虧冷卻

**作用：** 連續虧損太多筆時，自動暫停交易一段時間，避免情緒化連續追單。

---

## core/pipeline/ — 資訊管線與排程

這是系統的「情報網」— 收集市場情緒和外部數據。

### `scheduler.py` — 排程引擎

**作用：** 管理所有定時任務。基於 APScheduler。

**註冊的任務：**

| 任務 | 頻率 | 說明 |
|------|------|------|
| WebSocket K 線 | 持續 | 即時數據流 |
| 資金費率 | 每 30 分鐘 | 監控多空情緒 |
| 爆倉監控 | 每 5 分鐘 | 偵測連環爆倉 |
| 恐懼貪婪指數 | 每 60 分鐘 | 市場情緒 |
| 幣種篩選 | 每 60 分鐘 | 分批掃描 |
| 策略評估 | 每 1 分鐘 | 信號檢查 |
| 持倉檢查 | 每 1 分鐘 | 止損/止盈觸發 |
| 心跳 | 每 10 分鐘 | 靜默，僅異常時通知 |
| 監控快報 | 每 60 分鐘 | 發 LINE 損益快報 |
| 每日報告 | 每天 0:00 UTC | 績效報告 |
| K 線轉存 | 每 6 小時 | 寫入 Parquet 檔 |

---

### `veto_engine.py` — 否決引擎

**作用：** 交易信號的「安全閥」。資訊管線只有否決權，不能觸發交易。

**否決條件：**

| 條件 | 做多被否決 | 做空被否決 |
|------|-----------|-----------|
| 恐懼貪婪指數 | ≥ 80 (極度貪婪) | ≤ 20 (極度恐懼) |
| 資金費率 | ≥ 0.001 (多頭擁擠) | ≤ -0.001 (空頭擁擠) |
| 爆倉異常 | ✅ 暫停所有 | ✅ 暫停所有 |
| 絞肉機行情 | ✅ 暫停所有 | ✅ 暫停所有 |

> **Testnet 模式下：** 恐懼貪婪和資金費率的否決會被放寬（`RELAX_VETO_ON_TESTNET=true`），只保留爆倉和絞肉機的否決。

---

### `fear_greed.py` — 恐懼貪婪指數

**作用：** 定期從 Alternative.me API 抓取加密貨幣恐懼貪婪指數(0-100)。
- 0~25 = 極度恐懼
- 75~100 = 極度貪婪

---

### `funding_rate.py` — 資金費率監控

**作用：** 從幣安抓取各幣種的資金費率。正費率=多頭付費給空頭（多頭擁擠），負費率=反向。

---

### `liquidation.py` — 爆倉監控

**作用：** 監控市場上的連環爆倉事件。大量爆倉=市場極端情緒，此時不宜開新倉。

---

## core/brain/ — 大腦狀態管理

### `state.py` — 大腦狀態

**作用：** 讀取/更新 `data/brain_state.json`，允許動態調整策略參數，無需重啟程式。

**可覆寫的參數：**

| 參數 | 預設 | 說明 |
|------|------|------|
| `adx_min` | 25.0 | ADX 門檻 |
| `rsi_oversold` | 30.0 | RSI 超賣線 |
| `rsi_overbought` | 70.0 | RSI 超買線 |
| `skip_on_chop` | true | 絞肉機時跳過 |
| `relax_veto` | false | 放寬否決 |

**機制：** 每 15 分鐘自動重載，帶 60 秒快取避免頻繁讀檔。

## core/data/ — 數據層

### `websocket_feed.py` — WebSocket 即時數據流

**作用：** 持續連線幣安 WebSocket，接收 10 個幣種 × 4 個時間框架 = 40 條即時 K 線數據流。

- K 線收盤時透過回調通知主系統觸發分析
