# TradingBrain V9 使用說明書

## 1. 系統概要

TradingBrain V9 是一套以 Binance Futures 為目標的自動交易系統，包含：

- Python 後端交易引擎
- FastAPI 控制與資料 API
- React/TypeScript 儀表板
- Telegram 通知
- SQLite 研究與交易資料庫

目前系統採用：

- 結構優先的進出場邏輯
- `4h / 1h / 15m / 1m` 多時間框架流程
- `soft stop + hard stop` 雙層保護
- `trend_following / breakout_retest / mean_reversion` 三策略

## 2. 環境設定

### 2.1 `.env`

至少需要確認：

```env
BINANCE_API_KEY=your_key
BINANCE_API_SECRET=your_secret
BINANCE_TESTNET=true
TRADING_MODE=paper

TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

DASHBOARD_USERNAME=admin
DASHBOARD_PASSWORD=replace_with_a_strong_password
SECRET_KEY=replace_with_a_long_random_secret
```

### 2.2 模式說明

- `TRADING_MODE=paper`
  - 純模擬交易
- `BINANCE_TESTNET=true`
  - Binance Futures Demo / Testnet
- `BINANCE_TESTNET=false` 且 `TRADING_MODE=live`
  - 實盤

## 3. 啟動方式

## 4. 交易邏輯

### 4.1 市場分類

- `TRENDING`
  - 主要給 `trend_following`、`breakout_retest`
- `RANGING`
  - 主要給 `mean_reversion`
- `TRANSITIONAL`
  - 允許部分策略降權運行，不再直接卡死整套系統

### 4.2 多時間框架

- `4h`
  - 高週期方向背景
- `1h`
  - 方向過濾與補充信心
- `15m`
  - setup 產生
- `1m`
  - 進場觸發 / retest confirm

### 4.3 三套策略

- `trend_following`
  - 順勢回踩續行
- `breakout_retest`
  - 結構突破回踩後確認進場
- `mean_reversion`
  - 區間邊界回歸

## 5. 止損與止盈

### 5.1 雙層止損

- `soft stop`
  - 交易邏輯失效
  - 需收盤確認，不是 wick 掃到就走
- `hard stop`
  - 災難保命線
  - 用於極端行情、系統異常、斷線等情況

### 5.2 結構區域

系統不再只依賴單一 stop price / tp price，而會同時記錄：

- `stop_zone_low / stop_zone_high`
- `tp1_zone_low / tp1_zone_high`
- `tp2_zone_low / tp2_zone_high`
- `tp3_zone_low / tp3_zone_high`

### 5.3 三策略出場哲學

- `trend_following`
  - 給趨勢較大的呼吸空間
- `breakout_retest`
  - 以 retest 結構是否失敗為核心
- `mean_reversion`
  - 偏短打，較快兌現

## 6. Telegram 通知

目前通知統一走 Telegram，不再走 LINE。

### 6.1 會通知的事件

- 系統啟動
- 每小時監控快報
- 開倉
- TP / SL / trailing 出場
- 風控攔截
- 每日報告

### 6.2 測試通知

如果是測試或 smoke test，訊息會標示為：

- `🧪 TradingBrain V9 交易測試`

不會再偽裝成正式開倉通知。

## 7. 排程

### 7.1 常駐任務

- 持倉檢查
- 心跳檢查
- 每小時監控快報
- 大腦熱重載
- 每日報告

### 7.2 每日報告

- 以 `Asia/Bangkok` 為主時區
- 每日 `00:00` 產出
- 會寫入：
  - `logs/daily_reports`
  - `logs/daily_reports/agent_reports`

## 8. 前端顯示

### 8.1 Dashboard / Trades

前端目前會顯示：

- `soft stop`
- `hard stop`
- `stop zone`
- `TP1 / TP2 / TP3`

避免把 V7 的雙層保護誤看成舊版單線止損。

### 8.2 Risk 頁面

Risk 頁已對齊目前 preset：

- `conservative`
- `moderate`
- `passive_income`
- `aggressive`
- `training`

## 9. 重要資料路徑

- 主程式：
  - `main.py`
- 風控設定：
  - `config/risk_defaults.json`
- 資料庫：
  - `data/trading_brain.db`
- 日誌：
  - `logs/`
- 每日報告：
  - `logs/daily_reports/`
- 自動分析報告：
  - `logs/daily_reports/agent_reports/`

## 10. 維護原則

- 先看結構，再看 K 線，再看指標
- 槓桿是結果，不是固定值
- 不要把測試通知混成正式通知
- 不要讓文件與前端顯示落後於後端邏輯
